## 2. 25-04 토픽 기기 간 동기화 실패 문제 해결 (SQL 쿼리 오류 수정)
* **원인**: 
  * PC에서 푼 데이터를 핸드폰으로 새로고침할 때, 핸드폰의 로컬스토리지에 기존 세션 ID가 없으면 백엔드 서버가 DB(`app_session` 테이블)에서 가장 최신 세션을 자동 연동해주는 폴백 로직이 실행됩니다.
  * 하지만 해당 SQL 쿼리가 `ORDER BY id DESC` 형식으로 되어 있었으나, 실제 `app_session` 테이블 스키마에는 `id` 컬럼이 존재하지 않아 SQLite에서 쿼리가 완전히 크래시(실패)하는 현상이 있었습니다.
* **해결**:
  * `server/index.js` 내의 두 군데의 세션 조회 쿼리 정규식을 `ORDER BY id DESC` -> **`ORDER BY updated_at DESC`**로 변경하여, 쿼리 크래시 없이 실제 업데이트 일자 기준으로 가장 최신의 PC 풀이 데이터를 핸드폰으로 완벽하게 반환하도록 정정했습니다.

## 3. 락스크린 퀴즈 후보군 전체 확장 (해결방안 A 고도화)
* **해결**:
  1. `slice(0, topicLimit)` 처럼 일부 토픽만 골라내던 필터링을 **완전히 제거**하고, 데이터베이스에 저장된 **전체 토픽**을 후보군으로 삼아 LLM에 함께 전달하도록 로직을 변경했습니다.
  2. 토픽 본문 텍스트 로딩 시 기존의 동기식 `for...of` 루프 대신 **`Promise.all` 비동기 병렬 처리** 방식으로 교체하여, 전체 토픽 로드 시에도 수밀리초 내에 응답이 완료되도록 획기적으로 최적화했습니다.
  3. 최근 출제 질문 제외 목록(`recentQuestions` 캐시) 및 높은 변동성 설정(`temperature: 0.9`)과 완벽히 상호작용하여 다양한 문항이 전 범위에서 골고루 안배되어 출제됩니다.

## 4. 복습하기 답안 실시간 기기 연동 및 새로고침 동기화 패치
* **해결**:

## 5. 락스크린 문제 생성 실패 및 타임아웃 문제 해결 (API & 지침 맵핑 수정)
* **원인**: 
  1. **지침 누락**: `generateDailyLockscreenQuestions` 함수의 인자로 `lockscreenInstructionsPrompt`가 전달되었으나, 정작 프롬프트 생성용 템플릿(System/User Prompt)에는 해당 변수가 전혀 주입되지 않고 누락되어 있어 AI가 사용자 맞춤형 락스크린 지침을 읽지 못하고 오작동하는 심각한 논리 오류가 있었습니다.
  2. **Vercel 타임아웃 (504)**: Vercel 서버리스 환경은 **10초**의 매우 타이트한 실행 한도(Hobby plan)를 가집니다. 하지만 기존 `callLLMWithFailover` 로직은 Primary Key 실패 시, 불필요하게 429 오류에 대해 **지수 백오프 대기(sleep delay)**를 수행하고, 존재하지 않거나 과부하 상태인 12개 이상의 백업 모델들(`gemini-2.5-flash`, `gemini-3.5-flash` 등)을 연달아 순회하여 시간이 초과되는 근본적인 병목이 있었습니다.
* **해결**:
  1. **프롬프트 맵핑 정상화**: `systemInstruction` 템플릿 하단에 `[출제 지침 기준 (Lockscreen Generation Standards)]` 헤더와 함께 `${lockscreenInstructionsPrompt}` 변수를 정상적으로 주입 및 출력되도록 수정 완료했습니다.
  2. **Vercel 최적화 페일오버 설계**:
     * `isVercel` 환경 변수 감지 시, 429 할당량 초과 에러에 대해 **지수 백오프 대기 시간(sleep)을 0초로 스킵하고 즉시 다음 키/모델로 페일오버**하도록 수정하여 API 응답 속도를 획기적으로 개선했습니다.
     * `executionList` 생성 시, 루프를 키(Key) 기준으로 먼저 그룹화하여, preferred model인 `gemini-3.1-flash-lite`와 검증된 실 운영용 `gemini-1.5-flash`를 핵심 백업으로 순차적으로 우선 배치하였습니다. 불필요하고 실패율이 높은 백업 모델 탐색 단계를 완전히 줄임으로써 10초 제한 시간 내에 안정적으로 퀴즈를 반환하도록 조치했습니다.

## 6. AI 주제 개요 생성 ('개' 버튼) 및 저장/관리 기능 추가 (2026-06-30 패치)

* **배경 & 목적**:
  * 사용자가 학습 중에 특정 토픽에 대해 전체적인 맥락과 핵심 요약을 원할 때, AI 튜터가 구조화된 고품질 개요 및 시험 대비 가이드를 한 번의 클릭으로 즉시 작성해 주도록 기능을 신설했습니다.
  
* **상세 구현 사항**:
  1. **백엔드 (Vite Server)**:
     * `server/plugins/overviewsPlugin.js` 신설: AI 튜터에게 주제별 핵심 개념, 중요 공식/공식 유도, 주요 암기사항, 그리고 전공 시험 대비 꿀팁을 LaTeX와 Markdown 혼용 형식으로 작성하도록 하는 전용 프롬프트 생성 로직 탑재.
     * `GET /api/session/overviews` 및 `POST /api/session/overviews` API 엔드포인트 개설: 사용자별 커스텀 개요 리스트를 SQLite DB (`app_session` 테이블) 내 `formula_overviews` 레코드로 영속 보존 및 실시간 동기화.
     * `/api/chat` 라우트에 `overviewMode` 감지 로직 및 전용 핸들러 바인딩.
  2. **프론트엔드 (Vite Client)**:
     * **네 군데의 AI 도구 버튼 바 확장**: PC/모바일 학습 화면 및 주관식 퀴즈 뷰의 기존 "두" (두문자) 버튼 왼쪽에 붉은 장미색 테마의 **"개" (개요) 버튼** 추가.
     * **주제 개요 팝업 (Overview Prompt Modal)**: "개" 버튼 클릭 시 입력 창이 열리고, 원하는 임의의 주제나 키워드를 입력해 AI 생성 요청 가능.
     * **필수공식 리스트 탭 확장**: "공식", "표", "앞글자" 탭 옆에 **"개요"** sub-tab 추가.
     * **개요 데이터 렌더러**: `LatexRenderer`를 완벽히 연동하여 AI가 도출한 개요 마크다운과 복잡한 수학 공식(LaTeX)이 고해상도 수식 컴포넌트로 자동 렌더링되도록 구현.
     * **상태 관리 & Pull-to-refresh**: 개요 새로고침 API, 개요 카드 접기/열기 상태 보존, 더블클릭을 통한 개요 제목(Title) 수정 기능, 개별 카드 삭제 기능 추가 및 모바일 풀투리프레시(Pull-to-refresh) 리스트에 연동.

## 7. 믹스 복습(오늘의 필수 표/앞글자 믹스 복습) AI 재출제 먹통 오류 수정 (2026-06-30 패치)

* **원인**:
  * 믹스 복습(`mixed_acronym_table`)은 실제 데이터베이스의 `topics` 테이블에 행(row)이 존재하는 일반 토픽이 아니라, 사용자가 보관한 표와 앞글자를 조합하여 로컬에서 동적 생성하는 가상 토픽입니다.
  * 기존 `handleRefreshReviewQuestions` (AI 재출제 버튼의 클릭 핸들러)는 이를 구분하지 않고 서버의 `/api/topics/mixed_acronym_table_schedule/ai-questions` API로 재생성 요청을 무조건 전달했습니다.
  * 서버에서는 존재하지 않는 가상 토픽 ID에 대해 404/500 에러를 반환하였고, 프론트엔드는 이 에러로 인해 응답 대기 상태(로딩 상태)에서 탈출하지 못하고 먹통(Freeze)되는 현상이 발생했습니다.

* **해결**:
  * `client/src/App.jsx` 내 `handleRefreshReviewQuestions` 함수 진입부에 `mixed_acronym_table`에 대한 예외 처리 분기 로직을 구현했습니다.
  * 믹스 복습 토픽인 경우, 서버의 AI 질문 생성 API 대신 `handleOpenAIQuestions`와 동일한 방식으로 로컬 데이터베이스의 표/앞글자 풀을 불러옵니다.
  * 단, 재생성 목적에 부합하도록 매번 **새로운 무작위 시드(Random Seed)**로 셔플을 유도하여 사용자가 클릭할 때마다 5개의 복습 세트가 무작위로 새로 채워지며, 기존 입력/답안/세션 정보를 완전 초기화한 뒤 서버 세션에 다시 등록하도록 설계하여 원천적인 먹통 현상을 해결했습니다.

## 8. 표 내보내기 제목 파서 개선, 보관함 표 행 삭제 기능 추가 및 화면 먹통 버그 수정 (2026-06-30 패치)

* **상세 구현 사항**:
  1. **표 내보내기 제목 자동 정제 (markdownTableRenderer.js)**:
     * **문제**: 마크다운 본문에서 표 바로 윗줄을 제목 후보로 삼을 때, 줄 전체가 긴 설명글(예: `**주파수별 기준**: 진동의 주파수(Hz)가 낮을수록...`)인 경우 전체 문장이 표 제목으로 들어가 복잡해지는 문제가 있었습니다.
     * **개선**: 제목 후보 줄에 콜론(`:` 또는 `：`)이 포함되어 있고 그 앞의 텍스트가 40자 이하의 적합한 제목 후보이면 콜론 앞부분만 발라내고 markdown 볼드 기호(`*`)를 제거한 깔끔한 타이틀(예: `주파수별 기준`)만 추출하도록 개선했습니다. 또한, 전체 제목 길이를 최대 40자로 제한했습니다.
  2. **보관함 표 행 개별 삭제 기능 추가 (App.jsx)**:
     * **문제**: 저장된 비교표의 특정 행(Row)만 필요 없어져서 지우고 싶어도 개별 행 수정이 불가능했습니다.
     * **해결**: "표" 보관함 탭의 렌더링 방식을 기존 정적 HTML(`dangerouslySetInnerHTML`) 주입 방식에서, `parseHtmlTable`로 파싱하여 리액트 컴포넌트로 렌더링하는 동적 구조로 개편했습니다. 
     * 표의 가장 우측에 **[삭제]** 열을 신설하고 각 행마다 휴지통(Trash2) 버튼을 부착하여, 해당 행을 클릭해 손쉽게 삭제할 수 있게 했습니다. 삭제 시 남은 행 정보를 HTML 형식으로 실시간 재조립(`rebuildTableHtml`)한 후 데이터베이스에 즉시 동기화해 영속 저장합니다.
  3. **화면 먹통 오류 해결 (expandedOverviewIds 정의 누락 수정)**:
     * **원인**: 개요 탭을 렌더링할 때 개요 카드의 열고 닫힘 상태를 제어하는 `expandedOverviewIds` 상태 변수의 React `useState` 선언이 누락되어 `ReferenceError`가 발생해 화면 렌더링 전체가 크래시 및 먹통이 되었던 것이었습니다.
     * **해결**: `client/src/App.jsx`에 `const [expandedOverviewIds, setExpandedOverviewIds] = useState({});` 상태 변수를 추가 선언하여 크래시와 화면 먹통 현상을 완벽히 해결했습니다.

## 9. 실시간 튜터 대화 맥락 기반 AI 표 정제/작성 기능 추가, 개요 마크다운 표 구성 및 학습 팁 영역 제거 (2026-06-30 패치)

* **상세 구현 사항**:
  1. **실시간 튜터 대화 맥락 기반 AI 표 정제/작성 기능 (suggest-title-and-refine)**:
     * **문제/요구**: 실시간 튜터 창에서 공부하다가 생성된 적당한 표를 내보낼 때, 대화 맥락을 감안한 맞춤형 AI 작성이 요구되었습니다.
     * **해결**: 
       - 백엔드에 `/api/table/suggest-title-and-refine` POST API 엔드포인트를 신설하여 표 원본 HTML과 실시간 튜터 대화 맥락(최근 대화 기록)을 분석하도록 구축했습니다.
       - AI가 대화 주제에 걸맞은 가장 적절하고 전문적인 표준 표 제목(Title)을 추출하고, 내용에 기술사 표준 용어와 오탈자 수정을 가해 정제된 `<table>` HTML 구조로 자동 재작성합니다.
       - 프론트엔드에서 내보내기 버튼 클릭 시 로딩 스피너 및 AI 분석 인디케이터가 포함된 프리미엄 모달을 노출하며, AI 분석 결과를 표 제목 입력칸과 내용에 즉시 자동 완성해 줍니다.
  2. **개요 탭 구조 변경 (overviewsPlugin.js)**:
     * **요구**: 개요 내용을 마크다운 표(1열: 구분 [개요(40~60자), 메커니즘, 직관적 의미], 2열: 내용, 총 4개행) 형태로 구성합니다.
     * **해결**: 
       - `overviewsPlugin.js` 내 기본 개요 데이터를 상기 규격(구분/내용 2열 구성, 개요/메커니즘/직관적의미 4개행)의 마크다운 표 문자열로 전면 업데이트했습니다.
       - 개요 생성 AI 프롬프트(`generateOverviewTutorResponse`)를 수정하여 주제어 입력 시 사족 없이 반드시 4행 규격 마크다운 표 하나만 반환하도록 강제했습니다.
  3. **학습 팁 영역 제거 (App.jsx)**:

## 11. 주관식 서술형/앞글자 퀴즈 타이핑 랙 제거 및 API 채점 전 실시간 답안 버퍼 동기화 (2026-07-04 패치)

* **상세 구현 사항**:
  1. **주관식 퀴즈 컴포넌트 실시간 입력값 갱신 최적화**:
     * **문제**: 앞글자 퀴즈(`AcronymQuiz`) 및 주관식 퀴즈(`TableQuiz`) 렌더링 시, 텍스트가 입력될 때 발생하는 React 렌더 루프 타이핑 랙을 제거하기 위해 `<BufferedTextarea>`와 `<BufferedInput>`을 도입했었으나, 앞글자 입력 셀 등에 `onKeystroke` 콜백이 누락되어 있어 실시간 데이터 동기화 처리가 온전하지 못했습니다.
     * **해결**: 앞글자 퀴즈의 글자 입력 인풋 및 암기단어 입력 텍스트 에어리어에 `onKeystroke` prop을 연결하고 상위 컴포넌트의 `tableAnswersRef`를 올바르게 주입하여, 타이핑 시 React 전체가 리렌더링되지 않으면서도 입력된 최신 텍스트 정보가 실시간 버퍼 참조 레프(`tableAnswersRef.current`)에 완벽하게 갱신되도록 개선했습니다.
  2. **서술형(주관식 단답/개요 포함) 퀴즈의 AI 채점 전 버퍼 싱크 오류 수정**:
     * **문제**: 일반 주관식 문제 및 시험 모드(Exam Mode)의 서술형 퀴즈 입력 텍스트 영역(`<BufferedTextarea>`)에서 사용자가 답안을 입력하고 곧바로 [제출하고 채점하기] 버튼을 누를 경우, 입력 창 포커스가 빠지기(Blur) 전에 API 호출이 선행되어 최신 글자 몇 자가 반영되지 않은 채 구형 입력값으로 AI 채점이 진행되는 버그가 있었습니다.
     * **해결**: 일반 복습 및 시험 모드의 주관식 서술형 입력 창 모두에 `onKeystroke` 콜백을 연동하여, 키를 한 자 한 자 입력할 때마다 `tableAnswersRef.current` 및 `examTableAnswersRef.current` 참조 레프에 텍스트를 실시간 동기화했습니다. 이로 인해 마우스 클릭이나 단축키 전송 시점에 상관없이 언제나 최종 버퍼에 적재된 최신 답안으로 안전하게 채점이 진행됩니다.
  3. **코드베이스 내 핵심 구문 에러 및 중복 정의 제거**:
     * **문제**: 이전 코드 변경 중 `getSelectionTextWithLatex` 헬퍼 함수의 닫는 중괄호(`};`)가 누락되어 전체 `App.jsx` 하단 컴포넌트 전체가 해당 함수 본문에 중첩되는 끔찍한 구문 오류(Syntax Error)가 발생해 있었습니다. 이로 인해 빌드 도구(esbuild)가 `export default function App()` 구문을 함수 내부 선언으로 오인하여 `Unexpected "export"` 에러를 내며 빌드가 완전히 실패하고 있었고, `BufferedInput`과 `BufferedTextarea`가 모듈 스코프와 중첩 스코프에서 중복 선언되어 충돌하던 치명적인 결함이 있었습니다.
     * **해결**: 누락되었던 `getSelectionTextWithLatex` 함수의 닫는 괄호를 올바르게 추가하고, 중복되어 빌드 에러를 유발하던 1300라인 부근의 `BufferedTextarea` 및 `BufferedInput` 정의를 안전하게 제거했습니다. 패치 후 Vite Production Build를 실행하여 빌드 타임 경고 및 컴파일 에러 없이 `0 errors`로 완벽하게 4초 만에 번들링이 완료됨을 확인하고 Git 리포지토리에 즉시 커밋 및 푸시하여 반영했습니다.
  4. **AI 개요/앞글자 생성 팝업(Draggable Modal) 드래그 및 초기 위치 제어 랙/ReferenceError 해결**:
     * **문제**: 사용자가 학습 중 "개" (개요) 또는 "두" (두문자) 버튼을 누르면 AI 팝업창이 뜰 때, 최근 리팩토링된 드래그 이동 관련 핸들러 `handleGeneratorPopupMoveStart`와 이동 위치 저장용 상태 변_수 `generatorPopupPos`가 선언되지 않아 `ReferenceError: handleGeneratorPopupMoveStart is not defined` 오류가 나며 화면이 완전히 먹통(Freeze)되던 치명적인 버그가 발견되었습니다.
     * **해결**: 드래그 이동 팝업 핸들러인 `handleGeneratorPopupMoveStart`를 정상 정의하고, 로컬스토리지 연동형 `generatorPopupPos` 상태 변수를 생성하여 마우스 및 모바일 터치 드래그 위치 제어 기능이 크래시 없이 안정적으로 작동하도록 전면 보완했습니다. 마찬가지로 커밋 및 즉시 원격지에 배포를 완료하였습니다.
  5. **AI 생성기(두문자, 개요) 더블 클릭 중복 제출 방지 및 즉시 모달 닫기, 보관함 이동 팝업 구현**:
     * **문제**: 사용자가 앞글자(두문자) 및 개요 생성 팝업에서 단어/주제를 입력하고 엔터를 누르거나 [생성하기] 버튼을 중복 클릭할 경우, API 로딩이 완료될 때까지 입력 창이 닫히지 않고 열려 있어서 동일한 아이템이 2개씩 중복 등록되는 현상이 발생했습니다.
     * **해결**: 엔터를 치거나 생성하기 버튼을 누르는 순간 즉시(API 요청 시작 전) 입력창 모달을 닫아버리고(`setShowAcronymPromptModal(false)`, `setShowOverviewPromptModal(false)`) 입력 버퍼를 초기화하여 중복 제출 가능성을 원천적으로 차단했습니다. 모달이 닫힌 직후, "앞글자/개요 보관함 탭으로 이동하시겠습니까?" 라는 확인 팝업(`window.confirm`)을 띄워 사용자가 확인을 누를 경우 해당 보관함 탭(`acronym` 또는 `overview`)으로 자동 전환되게 하였으며, 탭 이동 시에는 백그라운드에서 실시간으로 생성 중인 스켈레톤 카드를 즉시 확인할 수 있도록 연동했습니다. 추가로 앞글자 생성기 인풋 필드들에도 `onKeyDown` 엔터 단축키 리스너를 매핑하여 부드럽고 직관적인 사용성을 제공합니다.
